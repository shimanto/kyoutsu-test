## ロゴ・アイコン体系

### デザインコンセプト

**3x3ヒートマップグリッド** をロゴマークとし、「弱点(赤) → 成長(黄) → 習得(緑)」の学習プログレスを視覚化。
配点ヒートマップという本サービスの核心機能をそのままブランドシンボルにしている。

配色 (上→下):
1. Row 1: `#dc2626` → `#f87171` → `#fde68a` (弱点ゾーン)
2. Row 2: `#fca5a5` → `#bef264` → `#4ade80` (成長ゾーン)
3. Row 3: `#86efac` → `#22c55e` → `#15803d` (習得ゾーン)
4. Bottom: `#22c55e` アクセントバー

### ファイル一覧

| ファイル | サイズ | 用途 | デザイン |
|---------|--------|------|---------|
| `favicon.svg` | 32×32相当 | ブラウザタブ・ファビコン | 3x3グリッドのみ (テキストなし) |
| `icon.svg` | 512×512 | メインアイコン・PWA | 3x4グリッド + 「大学物語」+ タグライン |
| `icon-192.png` | 192×192 | PWA・Android | icon.svg のPNG版 |
| `icon-512.png` | 512×512 | PWA splash・ストア | icon.svg のPNG版 |
| `og-image.svg` | 1200×630 | OGP (SNSシェア) | 左:グリッド + 右:テキスト + 機能一覧 |
| `og-image.png` | 1200×630 | OGP PNG版 | og-image.svg のPNG版 |

### コンポーネント

| コンポーネント | パス | 用途 |
|--------------|------|------|
| `LogoMark` | `apps/web/src/components/brand/LogoMark.tsx` | アプリ内で使う再利用可能なロゴマーク。`className` でサイズ変更可能 |

### 統一ルール

- **アプリ内ではemoji (📚) を使わない** — 必ず `LogoMark` コンポーネントを使用
- **テキストなしで使える** — favicon.svg のようにグリッドだけでもブランドが認識できる
- **配色は固定** — グリッドの赤→緑グラデーションは変更しない
- **背景色は `#030712`** (gray-950) — ダークテーマに統一

### アイコン・OGP PNG生成

```bash
node scripts/generate-icons.js
```

icon.svg → icon-192.png, icon-512.png, og-image.svg → og-image.png を一括生成。
sharp が必要 (`pnpm add -D sharp`)。代替: https://svgtopng.com/

## 公式タグライン

**「弱点が見える。だから伸びる。」**

> 選定理由: 課題→解決の構造で短く力強い。「弱点検出」という本サービスの核心機能を想起させ、
> 「物語」という名前だけでは伝わらない「学習ツール」としての価値を補完する。
> 詳細は [認知度調査レポート](brand-recognition-report.md) 参照。

### 拡張タグライン

| 形式 | テキスト | 用途 |
|------|---------|------|
| **短縮形** | 弱点が見える。だから伸びる。 | ロゴ横・ファビコン・OGPタイトル |
| **拡張形** | 弱点が見える。だから伸びる。— 共通テスト全9科目を可視化する学習マップ | LP Hero・広告バナー |

### 不採用候補 (参考)

| # | タグライン | 不採用理由 |
|---|-----------|-----------|
| 1 | 「共通テスト、全科目を一枚のマップで。」 | 機能説明的でエモーショナルさに欠ける |
| 3 | 「忘却曲線が味方になる受験勉強。」 | 忘却曲線を知らない人には意味不明 |
| 4 | 「あなたの900点、色で見える化。」 | 「900点」が共通テスト前提知識を要求 |

### ブランドボイス (トーン&マナー)

**トーン**: 自信を与える、簡潔で力強い言葉。受験生の不安に寄り添いつつ、科学的根拠で安心感を提供する。

**DO:**
- 課題→解決の構造で書く（例: 弱点が見える→だから伸びる）
- 具体的な数字を使う（9科目、900点、SM-2）
- 色・可視化のメタファーを活用する
- 受験生が主語の文を書く（あなたの弱点、あなたの学習マップ）

**DON'T:**
- 専門用語だけで説明しない（忘却曲線の説明なしに使わない）
- 煽りすぎない（焦らせるのではなく、自信を持たせる）
- 他サービスを否定しない
- 曖昧な表現を使わない（「すごい」「便利」等）

## 学習ステージ別コピー

定数定義: `packages/shared/src/branding.ts` の `STAGE_COPY` / `getStageCopy()`

ユーザーの全体正答率に応じて、ホーム画面に自動表示される励ましメッセージ。

| ステージ | 正答率 | headline | encouragement |
|---------|--------|----------|---------------|
| 基礎固め期 | 0-30% | まずは土台から。一歩ずつ、確実に。 | 積み上げた分だけ、マップの色が変わる。 |
| 成長期 | 31-60% | 伸びている。この勢いを止めるな。 | 赤が減り、緑が増えてきた。 |
| 仕上げ期 | 61-80% | あと一歩。精度を上げろ。 | ここからの1点が、合否を分ける。 |
| 完成期 | 81-100% | 仕上がった。自信を持って本番へ。 | マップが、あなたの実力を証明している。 |
